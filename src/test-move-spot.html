Setup: Player at "millbrook_square"
Action: Call GameFacade.MoveToSpot("millbrook_square")
Expected Result: Returns true (no-op success), no state change
// Test code var currentSpot = player.CurrentLocationSpot.SpotID; var result = GameFacade.MoveToSpot(currentSpot); Assert.IsTrue(result); Assert.AreEqual(currentSpot, player.CurrentLocationSpot.SpotID);

Setup: Player without required tokens/items for restricted spot
Action: Call GameFacade.MoveToSpot("noble_quarter")
Expected Result: Returns false, access denied message
// Test code (assuming noble_quarter has AccessRequirement) var result = GameFacade.MoveToSpot("noble_quarter"); Assert.IsFalse(result); // Player remains at original spot

Test 5: Null/Empty Spot Name

Setup: Player at any location
Action: Call GameFacade.MoveToSpot(null) or GameFacade.MoveToSpot("")
Expected Result: Returns false, "Invalid spot name" message
// Test code var result1 = GameFacade.MoveToSpot(null); var result2 = GameFacade.MoveToSpot(""); Assert.IsFalse(result1); Assert.IsFalse(result2);

Implementation Summary

Method Signature: public bool MoveToSpot(string spotName)

Key Features:
  • ✅ FREE movement (no attention cost) within same location
  • ✅ Validates spot exists in current location
  • ✅ Checks access requirements if present
  • ✅ Updates player.CurrentLocationSpot on success
  • ✅ Tracks spot discovery via AddKnownLocationSpot
  • ✅ Returns true on success, false on failure
  • ✅ Provides user feedback via MessageSystem
